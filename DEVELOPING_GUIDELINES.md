# Developing Guidelines

When working on the Banana-CongresoGT Site project, the following core rules must **always** be followed:

1. **Language (Code):** The project must be written completely in **English** (function names, variable names, comments, commit messages, database columns, etc.).
2. **Language (UI):** Any customer-facing string, content on the static pages, UI labels, or text meant for the final user must be exclusively in **Spanish**.
3. **Naming Conventions:** Avoid abbreviations for clarity. Use full words for variable and function names (e.g., `attendance_percentage` instead of `att_pct`, `congressman_id` instead of `c_id`).
